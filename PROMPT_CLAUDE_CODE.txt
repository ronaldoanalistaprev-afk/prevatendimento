### FLUXO CRÍTICO: WEBHOOK YCLOUD

Quando mensagem chega do WhatsApp:

1. **Receber** em `POST /api/webhooks/ycloud`
2. **Validar** signature do webhook (secret)
3. **Extrair:**
   - Número de origem
   - Instância (qual número recebeu)
   - Texto da mensagem
   - Timestamp
4. **Procurar contacto** por (número + instância)
   - Se não existe: CRIAR novo contacto como LEAD
   - Se existe: carregar dados
5. **Procurar conversa aberta** para esse contacto
   - Se não existe: CRIAR nova
   - Se existe: reutilizar
6. **Inserir mensagem** na conversa
7. **ATUALIZAR contacto + conversa:**
   ```
   ultima_resposta_quem = 'CLIENTE'
   data_ultima_msg_cliente = NOW()
   tempo_sem_resposta = 0
   status = 'ABERTA'
   ```
8. **Notificar dashboard** (WebSocket ou polling)

### ENDPOINTS PRINCIPAIS

**Webhooks:**
- `POST /api/webhooks/ycloud` → recebe mensagens

**Contactos:**
- `GET /api/contactos/:cpf` → detalhes + histórico
- `POST /api/contactos` → criar
- `PATCH /api/contactos/:id` → atualizar

**Conversas:**
- `GET /api/conversas/pendentes` → lista com filtros
- `GET /api/conversas/:id` → detalhes + mensagens
- `POST /api/conversas/:id/mensagens` → enviar (colaborador)

**Usuários:**
- `POST /api/auth/magic-link` → enviar link por email
- `POST /api/auth/verify` → verificar link
- `GET /api/usuarios/perfil` → dados do usuário

### FRONTEND - PÁGINAS PRINCIPAIS

1. **Login (Magic Link)**
   - Input email
   - Mensagem: "Link enviado para seu email"
   - Verificação automática

2. **Dashboard**
   - Métricas rápidas (conversas abertas, sem resposta, leads)
   - Lista de conversas pendentes (ordered por tempo sem resposta)
   - Cada linha mostra: cliente, última msg, tempo sem resposta, responsável
   - Clique abre conversa em detail

3. **Conversa Detail**
   - Dados do contacto no topo
   - Histórico de mensagens
   - Input para responder
   - Status da conversa

4. **Painel Gestor** (se role = ADMIN/GESTOR)
   - Conversas pendentes por colaborador
   - SLA violados (em vermelho)
   - Opção de atribuir conversas
